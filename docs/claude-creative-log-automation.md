# Claude作業ログからの創作進捗 自動記録機能

> Claude Desktop の推奨機能。過去のClaudeセッション履歴（`session_info`）を振り返り、その日に取り組んだ創作活動の進捗と作業内容を、生活管理ログ `logs/YYYY-MM-DD.md` へ半自動で記録する。センパイが手で書かなくても、Claudeで行った（またはClaudeで取得可能な）作業を継続的に記録できるようにするのが狙い。

共通仕様の正典は [AGENTS.md](../AGENTS.md)、ログ記録フォーマットの正典は [logs/README.md](../logs/README.md)。本ドキュメントはその上に乗る「Claude固有の自動記録運用」を定義する。

## 背景と目的

近ごろの創作作業の多くは Claude Desktop 上で行われている（VRMモデル制作、CreationsDB/CreationsAI のデータ整備、キャラ画像生成、リポジトリ保守 など）。これらは各セッションの履歴として残っているため、`session_info` ツールで読み返せば、「何に取り組み・どこまで進んだか」を機械的に振り返って記録に起こせる。

対象とするのは [logs/README.md](../logs/README.md) の推奨フォーマットのうち次の2欄のみ：

- **創作活動の進捗** … ナンバーテールズ／百花繚乱研究所の創作に直接関わる進捗。
- **取り組んだタスク** … それ以外の Claude Desktop 全般の作業（他プロジェクトの相談・調べ物・環境設定・再就職活動 等）。

体調・気分・睡眠・思考記録などの**機微な健康情報はこの機能では一切扱わない**。それらはセンパイ本人／Health Sheet が書く欄であり、自動記録の対象外とする。

## 仕組み

1. 実行日の**前日**を対象日 `YYYY-MM-DD` とする。
2. `session_info` の `list_sessions` / `read_transcript` で対象日のClaudeセッションを特定し、各作業を1〜2文で要約する（事実のみ、推測で盛らない）。
3. 各項目を「創作活動の進捗」「取り組んだタスク」に振り分ける。
4. `logs/YYYY-MM-DD.md` の **creative-log マーカー区間だけ**を編集する。

### creative-log マーカー

自動記録は、次のマーカーで囲まれた区間だけを対象にする（Health Sheet の `health-sheet` マーカーと同じ設計思想）。

```markdown
<!-- creative-log:start -->

## 創作活動の進捗

- 〜〜〜。［セッションタイトル］

## 取り組んだタスク

- 〜〜〜。［セッションタイトル］

<!-- creative-log:end -->
```

- マーカー区間が既にあれば、その中身だけを新しい内容で**置換**する（冪等・重複しない）。
- マーカーが無い既存ファイルには、`## 100(モモ)からのひとこと` 見出しがあればその**直前**に、無ければ**末尾**にマーカー付きで挿入する。
- ファイルが存在しなければ新規作成する（先頭に `# YYYY-MM-DD` 見出し。体調・気分・睡眠の欄は作らない）。
- 対象日に該当セッションが無ければ各欄に「（該当なし）」と書く。

### 安全則（厳守）

- `<!-- health-sheet:start -->` 〜 `<!-- health-sheet:end -->` の区間、およびセンパイの手書き内容（体調・気分・睡眠・思考記録・トピック・夜の追記・ひとこと 等）には**絶対に触れない**。
- 編集対象は `creative-log` マーカー区間のみ。
- `logs/*.md` は機微情報を含むためリポジトリ管理外（[.gitignore](../.gitignore)）。本機能はローカルファイルだけを編集し、コミットやpushは行わない。

## 運用（日次スケジュールタスク）

Claude Desktop の「Scheduled」機能に、前日ぶんをまとめる日次タスクを登録して運用する。

- タスクID: `momo-creative-log-daily`
- スケジュール: 毎日 07:30 頃（前日の作業を翌朝にまとめる。時刻はセンパイの生活リズムに合わせて調整可）
- 実体: `C:\Users\<user>\Claude\Scheduled\momo-creative-log-daily\SKILL.md`
- 動作: 上記「仕組み」の1〜4を実行し、100(モモ) の口調で要点を報告する。

スケジュールタスクはアプリ起動中に実行される。起動していない時刻に予定が来た場合は次回起動時に実行される。

### 手動での実行

任意のセッションで「昨日の創作進捗を記録して」「ここ数日の作業を棚卸しして」等と頼めば、同じ手順を手動で走らせられる。過去日をまとめて棚卸しする場合、古いセッションは `list_sessions` の取得上限から外れて拾えないことがある（その旨を注記する）。
