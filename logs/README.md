# 生活管理ログの記録フォーマット

このディレクトリには、「100(モモ)」との会話セッションで記録した体調・気分・生活習慣・CBT的な気づきなどを1日単位で保存します。個人の機微な健康情報を含むため、`logs/*.md`（本READMEを除く）は既定で git 管理対象外です（[../.gitignore](../.gitignore)参照）。

## ファイル名

`YYYY-MM-DD.md`（例: `2026-07-09.md`）

このファイル名形式（`YYYY-MM-DD.md`）の記録は、Claude連携ブリッジによってMisskey Bot（`../src/`）へ自動で取り込まれ、Botの応答文脈に利用されます（詳細は[../README.md](../README.md)参照）。

## bot-digest.md（自動生成・編集禁止）

`bot-digest.md` は、Misskey Bot側で保存された記録（日次チェックイン・思考記録・行動活性化・感謝日記）の直近14日分ダイジェストです。連携ブリッジ（Bot起動時・返信時・`npm run sync`）が自動生成・上書きするため、**手動で編集しないでください**。Claude（Desktop / Code）はセッション開始時にこのファイルを読み、Misskey側の記録を会話に反映します。

## 推奨フォーマット

```markdown
# YYYY-MM-DD

## 体調・気分

## 睡眠・生活習慣

## 創作活動の進捗

## 取り組んだタスク

## 100(モモ)からのひとこと / 次回への引き継ぎ
```

記録する・しない、内容を書き換えるかどうかの判断は、毎回センパイの意思を確認してから行うこと。

「取り組んだタスク」には、本リポジトリでの作業に限らず、Claude Desktop全体でそのセッション中に取り組んだこと・作業内容を簡潔に記す（他プロジェクトの相談、ちょっとした調べ物なども含めてよい）。「創作活動の進捗」は本リポジトリ・ナンバーテールズ創作活動に特化した進捗を記すもので、両者は役割が異なる（引き続き使い分けること）。

## Health Sheetアーティファクトによる直接書き込み

Claude Desktopのライブアーティファクト「Health Sheet」は、「logsに書き込む」ボタンで本ディレクトリのファイルを直接編集する。

- 日間シート → `logs/YYYY-MM-DD.md` ／ 週間シート → `logs/weekly-YYYY-MM-DD.md`（日付はその週の月曜）。
- 書き込みは **HTMLコメントのマーカー区間だけ**を置換する（初回はファイル新規作成、マーカーが無い既存ファイルには確認のうえ追記）。**マーカー外の手書き内容は保持される**ので、マーカー行自体を削除・編集しないこと。
- `weekly-*.md` は連携ブリッジの取り込み対象外（ブリッジは `YYYY-MM-DD.md` 形式のみ取り込む）。gitignoreの対象（`logs/*`）には含まれるため、git管理外である点は日次と同じ。

### logs/ で使うマーカー一覧（正典）

`logs/YYYY-MM-DD.md` には、複数の自動記録の担い手がそれぞれ自分の担当区間だけを安全に置換できるよう、HTMLコメントのマーカーを用いる。各マーカーは**自分の区間の中身だけ**を置換し、他人の区間・手書き内容には触れない。マーカー行（`<!-- ... -->`）自体は削除・改変しないこと。

| マーカー | 担当区間 | 書き手 |
| --- | --- | --- |
| `health-sheet:meds:start`〜`:end` | `## 服薬` の中身 | Health Sheet（日間・粒度別） |
| `health-sheet:mood:start`〜`:end` | `## 体調・気分` の中身 | Health Sheet（日間・粒度別） |
| `health-sheet:sleep:start`〜`:end` | `## 睡眠・生活習慣` の中身 | Health Sheet（日間・粒度別） |
| `health-sheet:tr:start`〜`:end` | `## 思考記録` の中身 | Health Sheet（日間・粒度別） |
| `health-sheet:memo:start`〜`:end` | `## 100(モモ)からのひとこと` の中身 | Health Sheet（日間・粒度別） |
| `health-sheet:start`〜`:end` | 健康欄（服薬・体調・睡眠等）をまとめて1区間 | Health Sheet（旧・一括形式） |
| `creative-log:start`〜`:end` | `## 創作活動の進捗` ＋ `## 取り組んだタスク` | Claude作業ログ自動記録（[../docs/claude-creative-log-automation.md](../docs/claude-creative-log-automation.md)） |

- **粒度別（`health-sheet:xxx`）が現行形式**。各見出しの直下に `<!-- health-sheet:mood:start -->` 〜 `<!-- health-sheet:mood:end -->` の形で置き、**見出しは区間の外・中身だけを区間の中**に入れる（例は [2026-07-18.md](./2026-07-18.md)）。
- **一括形式（`health-sheet:start`〜`:end`）は旧形式**で、過去ログ（2026-07-11〜07-17 等）に残る。健康欄をまとめて1区間で囲む。過去ログはそのまま有効。
- 健康欄（Health Sheet）と創作/タスク欄（`creative-log`）は書き手が別々なので、**同じ日のファイルに両方のマーカー区間が併存してよい**。互いの区間は決して重ねないこと（重なると双方の自動置換が壊れる）。

## 気分の数値記録について

「## 体調・気分」セクションでは、可能であれば最初の行に`気分: N/10`（Nは1〜10の整数）の形式で数値を残すと、将来のツール（週次・月次振り返りのグラフ化等）が数値を機械的に読み取れるようになる。センパイが数値化を望まない場合は無理に書かず、自由記述だけで構わない。

```markdown
## 体調・気分

気分: 7/10

- 朝は少し不安だったが、午後には落ち着いた。
```

## 睡眠・眠りの質の記録について

「## 睡眠・生活習慣」の眠りの質は、Health Sheetの**5段階**（5 ぐっすり〜1 あまり眠れず）を正典とする。過去ログや `bot-digest.md` に見られる「質 N/10」表記は10段階との誤解釈なので、**N/5基準で読む**こと（Bot→Claudeの[ダイジェスト生成](../src/bridge/digest-exporter.ts)も `/5` で出力する）。数値化を望まない場合は「ぐっすり／浅い／あまり眠れなかった」等の言葉でよい。

```markdown
## 睡眠・生活習慣

- 👁‍🗨 起床時刻: 07:00 ／ 💤 入眠 22:30・眠りの質 4（5段階）
```

## 服薬の記録について

[health-sheet-daily.md](../.cbt-datas/health-sheet-daily.md)の「## 服薬」セクションでは、以下のチェックボックス表記が機械可読な正典の書式（`scripts/extract-medication.mjs`が読み取る形式）となる。

```markdown
## 服薬

- 朝🌄: [x] 済 ／ 夜🌙: [x] 済
- 発作時⚡: 2回・頭痛時・少し楽になった
```

- 各スロット（朝🌄・日中☀️・食後🍽・夜🌙）は `[x]` で服用済み、`[ ]` で未服用、スロット自体を書かなければ「未報告」として扱われる（未服用と未報告は区別される）。
- 「発作時⚡」には飓服の回数・きっかけ・効き具合などを自由記述してよい。
- Misskey Botの`save_medication`ツールで保存された内容は`logs/bot-digest.md`の「## 服薬記録」セクションに反映されるため、`logs/*.md`側に必ず書く必要はない（気分の数値記録と同様、両方が独立した経路として併存する）。

## 任意セクション（CBTセルフケア機能を使った場合のみ）

[.cbt-datas/](../.cbt-datas/)配下のガイドに従ってCBTセルフケア機能を使った場合は、上記4見出しに加えて以下を必要に応じて追記してよい。使わない日は書かなくてよい。

```markdown
## 思考記録

## 行動活性化

## 感謝日記
```

- 「思考記録」には[.cbt-datas/thought-record.md](../.cbt-datas/thought-record.md)の7ステップの要点を簡潔にまとめる。
- 「行動活性化」には[.cbt-datas/behavioral-activation.md](../.cbt-datas/behavioral-activation.md)の計画内容と（あれば）実施後の振り返りを記す。
- 「感謝日記」には[.cbt-datas/gratitude.md](../.cbt-datas/gratitude.md)で挙げた3つの良かったことを記す。

## 記録のPDF出力

`logs/` の記録は `npm run export:pdf -- --from YYYY-MM-DD --to YYYY-MM-DD` で、期間指定のひとつのPDF（既定の出力先: `.cache/exports/`・git管理外）にまとめて出力できる。通院時に経過を見せる・振り返りを紙で読み返す等の用途を想定。Claude Desktop導入環境ではチャットで「100(モモ)」に頼むだけで生成できる。詳細は [../docs/logs-pdf-export.md](../docs/logs-pdf-export.md) を参照。
